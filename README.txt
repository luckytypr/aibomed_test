Отправляю тестовое задание для кандидата на вакансию backend разработчика (Python + Django).

Задание
 • Написать функционал для приложения — трекер задач. В приложении пользователи будут назначать задачи, следить за их статусом и обновлять данные. В случае изменения статуса задачи должны отсылаться уведомления наблюдающим за ней пользователям (в данном случае можно отсылать емейл). Также нужно следить за задачами и отсылать уведомление исполнителю, если задача просрочилась.

Предложенные модели
 • Задача
 • поля:
 • - название
 • - описание
 • - исполнитель
 • - наблюдатели
 • - статус (Планируется, Активная, Контроль, Завершена)
 • - время начала
 • - время завершения
 • - планируемое время завершения * дополнительно можно ввести в модель поддержку чек-листов
 • Изменение статуса
 • поля:
 • - предыдущий статус
 • - следующий статус
 • - задача
 • - кем изменен
 • Напоминание поля: - задача - текст напоминания - список пользователей для которых это напоминание
Технологии для использования:
 • Django
 • DRF
 • Celery
 • база данных PostgreSQL
 • аутентификация пользователей через JWT
